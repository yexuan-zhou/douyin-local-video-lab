# Local Video Lab 本地视频素材归档与AI复盘系统 V1.0 源代码打印整理指南

本文档用于从当前项目中整理软件著作权登记所需的源代码材料。正式提交前，请以登记平台或代理机构的最新要求为准。

## 一、适合作为自有代码的目录

建议优先从以下目录中整理源代码材料：

- `src/`：前端页面、组件、状态逻辑、业务流程等核心自有代码。
- `server/`：后端服务、接口逻辑、任务处理、报告生成等核心自有代码。
- `scripts/`：项目自有的导出、整理、辅助处理脚本。
- `docs/`：仅在需要说明材料结构时参考，一般不作为源代码页提交。
- `examples/`：如包含项目自有示例代码，可作为补充参考。

如某些目录在当前项目中不存在，可跳过，不需要为了凑页数创建无关代码。

## 二、不应包含的内容

整理源代码材料时，不建议包含以下内容：

- `node_modules/` 等第三方依赖目录。
- `.git/`、`.github/` 等版本控制或自动化配置目录。
- `downloads/`、`artifacts/`、`output/` 等运行产物或导出材料目录。
- 浏览器配置、缓存、个人本地配置文件。
- 第三方库源码、压缩后的构建产物、二进制文件。
- 包含账号、密码、Token、Cookie、密钥、真实手机号、身份证号、住址等敏感信息的文件。

## 三、前30页和后30页整理方法

1. 汇总自有源代码文件，优先选择能够体现核心功能的文件。
2. 按目录和文件名排序，保持代码顺序连续、可读。
3. 将整理后的代码放入一个文档中，例如 Word、WPS 或 Markdown 转 PDF。
4. 如果源代码超过60页，提交前连续30页和后连续30页。
5. 如果源代码不足60页，提交全部源代码。
6. 每页页眉建议写明：Local Video Lab 本地视频素材归档与AI复盘系统 V1.0。
7. 页码保持连续，便于审查人员核对。

建议前30页优先覆盖软件入口、页面组件、视频导入、关键帧抽取、摘要生成等核心流程；后30页优先覆盖报告导出、任务脚本、配置读取、错误处理等辅助流程。

## 四、隐私与敏感信息检查

导出源代码前，请至少检查以下内容：

- 搜索 `Token`、`Cookie`、`password`、`secret`、`key`、`账号`、`密码` 等关键词。
- 检查 `.env`、本地配置文件、浏览器配置文件是否被误加入。
- 检查是否出现真实身份证号、手机号、住址、邮箱等个人信息。
- 检查是否包含本地下载目录、个人用户名路径、临时缓存路径。
- 检查是否复制了第三方依赖源码。

如发现敏感信息，应先在用于提交的源代码副本中删除或替换为占位符，例如 `<申请人姓名>`、`<联系方式>`、`<开发完成日期>`。

## 五、转成 PDF 的方法

可任选以下方式导出 PDF：

1. 使用 VS Code 打开整理后的 Markdown 或代码文档，通过 Markdown PDF 插件导出 PDF。
2. 将代码复制到 Word 或 WPS，设置等宽字体、页眉和页码后另存为 PDF。
3. 使用浏览器打印功能，将整理好的 HTML 或 Markdown 预览页面打印为 PDF。
4. 使用命令行转换工具时，请先确认生成文件不包含第三方依赖、密钥和个人隐私。

推荐页面格式：

- A4 纸张。
- 等宽字体，例如 Consolas 或 Courier New。
- 字号 9 至 11 号。
- 页眉标注软件名称和版本号。
- 页脚标注页码。

导出后请打开 PDF 逐页检查，确认页面没有乱码、重叠、截断或敏感信息。
