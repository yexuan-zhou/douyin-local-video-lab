# Compliance and Acceptable Use Policy

This project is designed for creator-owned or authorized video material. It is a local archive, key-frame extraction, AI-assisted summary, and content review workflow. It is not a tool for unauthorized copying, platform rule evasion, or bulk content collection.

## Allowed Use

- Videos created by the user.
- Videos the user has explicit permission to process.
- Internal team media assets.
- Publicly licensed or legally reusable material.
- Local material organization, teaching, research, and review.

## Prohibited Use

- Downloading, copying, distributing, or commercializing other people's videos without authorization.
- Bypassing platform technical restrictions, DRM, login requirements, paywalls, or access controls.
- Bulk crawling platform content.
- Capturing, storing, or reusing cookies, tokens, account names, passwords, or private credentials.
- Using unauthorized video material for commercial services, ads, portfolios, training datasets, or client deliverables.
- Violating copyright, portrait rights, privacy rights, local law, or platform terms.

## Privacy

- The default workflow is local-first.
- The project does not require platform account names or passwords.
- The project must not save user private credentials, cookies, or tokens.
- User videos are not uploaded by default.
- If users later configure a third-party AI service, they are responsible for checking that service's privacy, retention, and data-processing terms.

## Developer Notes

- Do not add features that bypass access restrictions.
- Do not add simulated login, cookie extraction, token capture, credential storage, or stealth automation.
- Do not turn optional link handling into a crawler or mass collection workflow.
- All external link processing must keep authorization and compliance as explicit assumptions.
- README, examples, UI text, screenshots, and demos must avoid encouraging copyright infringement.
- Demo data must be fictional or clearly licensed for reuse.
